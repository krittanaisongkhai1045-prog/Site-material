# ระบบวัสดุไซต์งาน (เว็บ + มือถือ + ข้อมูลกลาง)

เว็บแอปติดตามวัสดุก่อสร้าง: สโตร์กลาง → เบิกไปแต่ละอาคาร, ใบเบิกแนบรูปเป็น PDF, รายงาน/กราฟอินโฟกราฟ, หลายโครงการ, เผื่อเก็บงาน %, นำเข้า Excel

แอปทำงานได้ **2 โหมด**:
- **โหมดในเครื่อง** (ค่าเริ่มต้น) — ไม่ต้องตั้งอะไร เปิดใช้ได้เลย ข้อมูลเก็บในเบราว์เซอร์เครื่องนั้น (แยกกันแต่ละเครื่อง)
- **โหมดคลาวด์ (แนะนำ)** — ล็อกอินจริงด้วยอีเมล/รหัสผ่าน + ทุกคนเห็น **ข้อมูลชุดเดียวกัน** แบบเรียลไทม์ (ต้องต่อ Supabase ครั้งเดียว)

---

## ส่วนที่ 1 — เอาขึ้นเว็บฟรีด้วย GitHub Pages (ใช้บนมือถือได้)

1. สมัคร/เข้า https://github.com → กด **New repository** ตั้งชื่อ เช่น `site-material` → Create
2. กด **Add file → Upload files** แล้วลากไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้นไป (index.html, manifest.json, sw.js, ไอคอน, .nojekyll ฯลฯ) → Commit
3. ไปที่ **Settings → Pages** → Source เลือก `Deploy from a branch` → Branch `main` / โฟลเดอร์ `/ (root)` → Save
4. รอสักครู่ จะได้ลิงก์เว็บ เช่น `https://ชื่อคุณ.github.io/site-material/`
5. เปิดลิงก์บนมือถือ → เมนูเบราว์เซอร์ → **เพิ่มไปยังหน้าจอโฮม** จะได้ไอคอนแอปเหมือนแอปจริง

> แค่นี้ก็ใช้งานบนมือถือได้แล้ว (แต่ยังเป็นข้อมูลแยกเครื่อง) ทำต่อส่วนที่ 2 เพื่อแชร์ข้อมูลกลาง

---

## ส่วนที่ 2 — เปิดล็อกอินจริง + ข้อมูลกลางแชร์กัน (Supabase ฟรี)

1. สมัคร/เข้า https://supabase.com → **New project** (ตั้งรหัสฐานข้อมูล เก็บไว้)
2. เมนู **SQL Editor** → วางเนื้อหาไฟล์ `supabase-schema.sql` → กด **Run** (สร้างตาราง + สิทธิ์ให้อัตโนมัติ)
3. เมนู **Project Settings → API** จะเห็น
   - **Project URL** (เช่น `https://abcdxyz.supabase.co`)
   - **anon public** key (คีย์ยาว ๆ)
4. เปิดไฟล์ `index.html` แก้ 2 บรรทัดใกล้ ๆ ต้นสคริปต์ (หา `SUPABASE_URL`):
   ```js
   const SUPABASE_URL='https://abcdxyz.supabase.co';   // ← ใส่ Project URL
   const SUPABASE_ANON_KEY='eyJhbGciOi...';            // ← ใส่ anon public key
   ```
5. บันทึกแล้วอัปโหลดทับขึ้น GitHub อีกครั้ง
6. เปิดเว็บ → จะมีหน้า **เข้าสู่ระบบ (คลาวด์)** ให้กด **สมัครสมาชิกใหม่** ด้วยอีเมล/รหัสผ่าน แล้วเข้าสู่ระบบ
7. คนอื่นในทีมสมัครบัญชีของตัวเอง → ทุกคนจะเห็น **ข้อมูลชุดเดียวกัน** และอัปเดตหากันแบบสด ๆ

> **การยืนยันอีเมล:** ถ้าไม่อยากให้ต้องยืนยันอีเมลก่อนใช้ ไปที่ Supabase → Authentication → Providers → Email → ปิด "Confirm email"

---

## ข้อควรรู้ / ข้อจำกัด
- **anon key ใส่ในหน้าเว็บได้ปลอดภัย** — ข้อมูลถูกป้องกันด้วย RLS (ต้องล็อกอินก่อนถึงเข้าถึงได้)
- ตอนนี้ทุกคนที่ล็อกอินเห็น **workspace เดียวกัน** (เหมาะกับทีมเดียว) — ถ้าต้องแยกหลายทีม/สิทธิ์ ค่อยต่อยอดได้
- การแก้พร้อมกันหลายคน **โครงการเดียวกัน** ระบบใช้ "บันทึกล่าสุดทับ" (last-write-wins) — เหมาะกับงานที่ไม่แก้ชนกันบ่อย
- รูปถ่ายถูกย่อและเก็บในฐานข้อมูล — ถ้าใช้รูปเยอะมาก ๆ แนะนำต่อยอดไปเก็บที่ Supabase Storage ภายหลัง
- ฟีเจอร์สร้าง PDF/รูป และนำเข้า Excel ต้องมีอินเทอร์เน็ต (โหลดตัวช่วยจาก CDN)
